# CI/CD Fundamentals

## Overview

CI/CD describes practices for integrating code frequently, verifying changes automatically, and moving releasable software toward users. This topic establishes the differences between continuous integration, continuous delivery, and continuous deployment, then introduces the parts of a pipeline and the feedback loops around it.

## Why It Matters

The terms are often used as if they mean the same thing. Clear distinctions make pipeline goals, automation boundaries, and human decisions easier to design. The fundamentals also explain why small changes, fast validation, and visible results reduce integration and release risk.

## Main Concepts

- Frequent integration and automated verification
- Releasable software and deployment automation
- Pipeline inputs, execution, outputs, and feedback
- Shared ownership and continuous improvement

## Learning Objectives

After completing this section, the learner should be able to:

- Distinguish continuous integration, delivery, and deployment.
- Identify the main parts and feedback paths of a pipeline.
- Explain how CI/CD practices support small, repeatable changes.
